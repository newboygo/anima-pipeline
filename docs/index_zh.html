功能

核心功能

专为Windows打造，兼顾移动端访问与扩展性。

AI
LLM标签自动生成
支持Gemini API、LM Studio以及任何兼容OpenAI的API。从角色名自然生成标签。
L4
LoRA最多4个插槽
带缩略图预览的卡片式LoRA选择界面。自动与工作流JSON同步。
CH
多角色管理
保存并切换角色预设。最多可同时配置6个角色。
NET
手机平板支持
同一WiFi下可从iPhone/iPad操作。ComfyUI需添加--listen参数。
UI
日/英/中 三语切换 v1.5.22
首次启动时自动检测系统语言。可在日文、英文、中文之间随时切换。
LOG
日志脱敏 v1.4.7
API密钥和敏感数据在日志中被遮蔽。支持安全导出ZIP用于分享或调试。
HIS
生成历史 v1.5.1
所有生成记录存入数据库。可重新打开过去的生成结果、微调提示词后重新生成。
PR
预设层级管理 v1.5.1
分离角色、场景、镜头、品质、LoRA及合成预设。可自由组合搭配。
DX
设置诊断工具 v1.5.1
内置自检UI可检测ComfyUI和LLM的连接问题。无需查看日志即可排错。
安装

5分钟快速上手

Windows用户只需双击 start_anima_pipeline.bat。

1
从ComfyUI导出工作流JSON
浏览 Templates > Anima > Save (API格式)，然后将文件放入 workflows/ 目录。
2
运行批处理文件
双击 start_anima_pipeline.bat。首次运行将自动安装依赖。
3
打开 localhost:7860
在浏览器中打开UI，设置ComfyUI地址和LLM配置，然后保存。
4
输入角色并开始生成
按Ctrl+Enter开始生成。结果会立即显示在界面中。

要求

所需环境

最低要求：ComfyUI + Python（系统Python或内置Python均可）。LLM为可选。

Python
3.10+（或内置）
如果内置了 python_embeded/python.exe，则无需系统Python。
ComfyUI
0.16.4 或更高版本
需要可用的Anima工作流。内置4个工作流模板（Anima1/2，带/不带LoRA x4）。移动端访问需添加 --listen --enable-cors-header。
LLM服务
可选
支持Gemini API / LM Studio / 任何兼容OpenAI的API。
不使用LLM也能工作 - 直接手动输入标签即可
